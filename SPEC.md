# MemoryLane — SPEC.md

## Concept & Vision

MemoryLane is a CLI tool that uses AI to automatically maintain a markdown journal of your coding work. It extracts context from git diffs, commits, and file changes to build a searchable daily log. The experience should feel like having a silent pair of eyes watching your work and chronicling it — effortless, automatic, and insightful. Developers fire it up once and never wonder "what was I working on last Tuesday?" again.

## Technical Design

### Stack
- **Runtime:** Node.js 20+ with ESM
- **Language:** TypeScript (strict mode)
- **Git integration:** simple-git / git log parsing
- **AI integration:** Ollama-compatible API (local LLM), with fallback to OpenAI/Anthropic cloud APIs
- **CLI framework:** Commander.js
- **Date handling:** date-fns

### Features & Behavior

#### `memory-lane init`
- Creates `.memorylane/` directory with config template
- Creates `.memorylanerc` (YAML) for API settings (ollama URL, model, etc.)
- Initializes `journal/` directory for daily markdown files

#### `memory-lane run` (default command)
- Scans git diff since last run (or today midnight)
- Extracts: commit messages, changed files, diff summary
- Sends context to LLM with prompt: "Summarize this coding session in 2-3 sentences as a work log"
- Appends to `journal/YYYY-MM-DD.md` with timestamp
- Tracks last-run timestamp in `.memorylane/last-run`
- **Non-destructive:** never overwrites, only appends
- **Idempotent:** re-running same day appends nothing or a small update

#### `memory-lane search <query>`
- Searches all journal files with simple regex + highlighting
- Outputs matching entries with dates

#### Config (`.memorylanerc`)
```yaml
provider: ollama  # or openai, anthropic
ollama:
  url: http://localhost:11434
  model: llama3
openai:
  apiKey: ...
  model: gpt-4o
journalDir: journal
excludePatterns:
  - "**/node_modules/**"
  - "**/.git/**"
```

### Acceptance Criteria

1. `memory-lane init` creates proper directory structure with config file
2. `memory-lane run` parses git state and generates a journal entry
3. Journal entries contain: date header, AI summary, file changes list
4. Re-running on same day does NOT duplicate entries (dedup by date)
5. Works offline with Ollama; has clear error if no LLM available
6. `memory-lane search` finds entries matching query
7. TypeScript compiles without errors
8. Basic unit tests pass for git parsing and journal writing

## Implementation Notes

- Use simple-git for git operations (lightweight, Promise-based)
- Ollama API: POST to `/api/generate` with prompt
