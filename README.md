# MemoryLane

**AI-powered CLI tool that automatically maintains a markdown journal of your coding work.**

MemoryLane watches your git activity and uses AI to generate daily summaries of what you worked on. Never wonder "what was I doing last Tuesday?" again.

## Features

- **Automatic git tracking** — scans commits and uncommitted changes since your last run
- **AI summarization** — uses local (Ollama) or cloud (OpenAI, Anthropic) LLMs to generate concise work summaries
- **Daily markdown journal** — creates organized, searchable journal entries in `journal/YYYY-MM-DD.md`
- **Smart deduplication** — re-running won't create duplicate entries for the same day
- **Multi-provider support** — works with Ollama, OpenAI, or Anthropic
- **Searchable** — find any entry by keyword with the built-in search command

## Installation

```bash
npm install -g memory-lane
```

Or build from source:

```bash
git clone https://github.com/your-username/memory-lane.git
cd memory-lane
npm install
npm run build
```

## Quick Start

```bash
# 1. Initialize in your project
memory-lane init

# 2. Configure your LLM provider (edit .memorylanerc)
# Default: Ollama at localhost:11434 with llama3 model

# 3. Run the journaler (do this daily or after significant work)
memory-lane run

# 4. Search your journal anytime
memory-lane search "authentication"
```

## Commands

### `memory-lane init`
Initializes MemoryLane in the current directory:
- Creates `.memorylane/` directory (stores last-run timestamp)
- Creates `journal/` directory (stores daily entries)
- Creates `.memorylanerc` configuration file

### `memory-lane run` (default)
Scans git activity since your last run and generates a journal entry:
- Finds commits since last run (or midnight today)
- Lists uncommitted file changes
- Generates AI summary of your session
- Appends to `journal/YYYY-MM-DD.md`

**Options:**
- `--date <YYYY-MM-DD>` — Backfill a journal entry for a past date. Useful when you forgot to run MemoryLane after working on a previous day.
- `--dry-run` — Preview what would be journaled without saving the entry.
- `--week` — Generate a weekly summary instead of a daily entry.

### `memory-lane search <query>`
Searches all journal entries for matching text. Case-insensitive.

### `memory-lane --help`
Shows all available commands.

## Configuration

Edit `.memorylanerc` to configure your AI provider:

### Ollama (default, local)
```yaml
provider: ollama
ollama:
  url: http://localhost:11434
  model: llama3
```

### OpenAI
```yaml
provider: openai
openai:
  apiKey: your-api-key
  model: gpt-4o
```

### Anthropic
```yaml
provider: anthropic
anthropic:
  apiKey: your-api-key
  model: claude-3-5-sonnet-20241014
```

### General settings
```yaml
journalDir: journal
excludePatterns:
  - "**/node_modules/**"
  - "**/.git/**"
```

## Requirements

- **Node.js 20+**
- **Git repository** (must be initialized in a git repo)
- **LLM provider** (Ollama running locally, or OpenAI/Anthropic API key)

### Ollama Setup (recommended for privacy)
```bash
# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull a model
ollama pull llama3

# Ollama runs at localhost:11434 by default
```

## Journal Format

Entries are saved as `journal/YYYY-MM-DD.md`:

```markdown
## 2026-04-22

**Summary:** Implemented OAuth2 PKCE authentication flow. Added session renewal logic and fixed token refresh race condition in the token manager.

**Changes:**
  - `src/auth/middleware.ts` — refactored OAuth2 handler
  - `src/auth/token-manager.ts` — added renewal logic with mutex lock
  - `tests/auth/token.test.ts` — added tests for refresh race condition

_Generated by MemoryLane at 22:45_
```
